Overview

I'm Philipp, a physics student from Munich, who likes building stuff. This site is mainly a collection of interactive demos I built and maybe a personal blogpost every now and then. The code for most of the projects is available on my GitHub page.

Square Lights

For the past few months I've been working hard to build this company. We are producing music reactive LED Panels, so if that sounds like something you're interested in, please check us out!

Optics

Exams are coming up, so I decided to code up a simulation for optical systems. It supports lenses and some basic mirrors.
All the rays are calculated according to the thin lens approximation and therefore get quite inaccurate for rays, that are distant from the optical axis.

Stupid Units

A simple website that converts weights, distances and energies into weird units, that are in the same order of magnitude

LED Matrix demo page

A startup idea I'm currently working on. Music reactive LED matrices, that don't just react to music, but rather build intricate custom lightshows to any song you want. The main project is being written in python on a private GitLab repo, but i ported an old version of the code to JavaScript for demo purposes Transcrypt.
Go to the website, log into Spotify and play a song to see what it can do!

Marching sqares

An implementation of the marching squares algorithm with interpolation.

Physics simulation

My attempt at implementing a spatial hashing algorithm for 2D physics. The algorithms uses circles, points and axis aligned rectangles as primitives and detects overlaps efficiently. Currently only simple spring repulsion and attraction forces are supported for the primitives. The solver is a semi implicit Euler implementation, I'll try to implement somthing more stable over time.

Electric fields

Inspired by the lecture we had, I made a small visualizer that shows how point charges interact with each other, move and most importantly shape the electric field. Click on the green canvas to add charges, modify the parameters, build parcours for them, go nuts! It's not exactly physically accurate, because I'm using discrete time steps, but it gets the job done and looks pretty cool. Could be improved by using the same Method as the solar system simulation.

Solar system orbits

Here's an attempt at simulating planetary orbits in our solar system. Since stability is key for such a simulation, this uses a Runge Kutta fourth order algorithm (RK4) to solve the differential equations. You can increase planetarymass by selecting the planets and using the = key or - key to decrease it. Planet textures by planetpixelemporium, the WEBGL library is three.js.

Corona Live Counter

A visualization of how often people get infected by corona/recover/die. The data is being fetched from this API and processed to calculate the average intervals. There's also a customizable version for wallpaper engine!

Corona Map

A variation of the live counter project that lights up the individual countries in regular intervals, according to the average time between infections in that country. It's not pretty, but it's functional

Virus simulation

Simple virus simulation to show the effects of social distancing, improved hygiene and different attributes of the virus. Just play around with the parameters and see what happens! And of course there's a customizable version for wallpaper engine once again.
