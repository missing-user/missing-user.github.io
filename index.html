Overview

My name is Philipp Jurašić, a physics student from Munich. Outside of university I love building electronics projects, websites and interactive simulations. Here's a selection of my recent projects to play around with, most of them are written in plain JS without any additional libraries or frameworks.

The code for all of the demos is available on my GitHub page.

Differential equations

Here we go with another maths heavy project. I tried my best to explain how the numerical methods work and add code examples where sensible. There are also a few demos solving popular differential equations like the Lotka Volterra equations or harmonic Oscillators.

Square Lights

For the past year my colleague and I've been working hard to build this company. We are producing music reactive LED Panels, so if that sounds like something you're interested in, please check us out!

Optics

Exams are coming up, so I decided to code up a simulation for optical systems. It supports lenses and some basic mirrors. All the rays are calculated according to the thin lens approximation and therefore get quite inaccurate for rays, that are distant from the optical axis.

Stupid Units

A simple website that converts weights, distances and energies into weird units, that are in the same order of magnitude

Marching squares

Marching squares is an algorithm to create boundaries around a point cloud. I use it to create these merging blobs, by implementing the algorithm with some linear interpolation.

Physics simulation

My attempt at implementing a spatial hashing algorithm for 2D physics. The algorithms uses circles, points and axis aligned rectangles as primitives and detects overlaps efficiently. Currently only simple spring forces are supported for the primitives. The solver is a semi implicit Euler implementation, so energy isn't perfectly conserved due to rounding errors

Electric fields

Inspired by the lecture we had, I made a small visualizer that shows how point charges interact with each other, move and most importantly shape the electric field. Click on the green canvas to add charges, modify the parameters, build parcours for them, go nuts! It's not exactly physically accurate, because I'm using discrete time steps, but it gets the job done and looks pretty cool. Could be improved by using the same Method as the solar system simulation.

Solar system orbits

Here's an attempt at simulating planetary orbits in our solar system. Since stability is key for such a simulation, this uses a Runge Kutta fourth order algorithm (RK4) to solve the differential equations. You can increase planetarymass by selecting the planets and using the = key or - key to decrease it. Planet textures by planetpixelemporium, the WEBGL library is three.js.

Corona Live Counter

A visualization of how often people get infected by corona/recover/die. The data is being fetched from this API and processed to calculate the average intervals. There's also a customizable version for wallpaper engine!

Corona Map

A variation of the live counter project that lights up the individual countries in regular intervals, according to the average time between infections in that country. It's not pretty, but it's functional

Virus simulation

Simple virus simulation to show the effects of social distancing, improved hygiene and different attributes of the virus. Just play around with the parameters and see what happens! And of course there's a customizable version for wallpaper engine once again.

Frustrated physicists
